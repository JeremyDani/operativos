## ATENCION: NO USAR CARACTERES ESPECIALES EN LOS NOMBRES DE LOS DIRECTORIOS O ARCHIVOS

## CREAR CARPETA E INGRESAR A ELLA

## CREAR ENTORNO VIRTUAL
virtualenv .venv

## CREAR ENTORNO VIRTUAL CON VERSION ESPECIFICA
virtualenv --python=/usr/local/bin/python3.10 .venv

## ACTIVAR ENTORNO VIRTUAL
source .venv/bin/activate

## INSTALAR DJANGO
pip install django

## INICIAR PROYECTO
django-admin startproject configuracion .

## PREPARAR MIGRACIONES
python manage.py makemigrations

## APLICAR MIGRACIONES
python manage.py migrate

## APLICAR MIGRACIONES DE CONEXION ESPECIFICA
python manage.py migrate --database=base_de_datos

## CREAR SUPERUSUARIO
python manage.py createsuperuser

## EJECUTAR SERVIDOR DE DESARROLLO
python manage.py runserver

## EJECUTAR UVICORN
uvicorn configuracion.asgi:application --reload

## CREAR ARCHIVO requiriments.txt
pip freeze > requirements.txt

# VACIAR ESQUEMA PUBLIC EN UN MODELO
python manage.py inspectdb > models.py

# INICIAR NUEVA APLICACION
python manage.py startapp nombre_aplicacion

## ACTUALIZAR TODOS LOS PAQUETES CON PIP3
pip list --outdated --format=freeze | grep -v '^\-e' | cut -d = -f 1  | xargs -n1 pip install -U

## ACTUALIZAR SOLO UN PAQUETE (No sus dependencias)
pip install -U --no-deps flask

## RECOLECTAR ARCHIVOS ESTATICOS
python manage.py collectstatic 

## INICIAR FRONTEND (Vite + Vue)
Desde la raíz del proyecto:

1) Instalar Node.js (recomendado Node 18+).

2) Entrar al directorio `frontend` e instalar dependencias (solo la primera vez o si cambian):

	cd frontend
	npm install

	(También puedes usar `yarn` si prefieres: `yarn install`)

3) Levantar el servidor de desarrollo (Vite):

	npm run dev

4) Construir para producción y previsualizar / servir:

	npm run build
	npm run preview    # previsualiza el build con Vite

	# Alternativa para servir la carpeta `dist`:
	npm install -g serve
	serve -s dist

Notas:
- Asegúrate de estar en el directorio del proyecto antes de ejecutar `cd frontend`.
- Si usas `pnpm` u otro gestor, reemplaza los comandos según corresponda.
